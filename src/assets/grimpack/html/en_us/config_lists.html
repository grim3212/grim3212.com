When looking through the configuration files you may come across a property that looks like this. 

S:"Blocks allowed when restrict powerstaff is active" <
    dirt
    grass
    cobblestone
    stone
>

This corresponds to a list of item or blocks depending on the context. You can use basic registry names so minecraft:dirt or grimpack:repulsor. But you can also do some special things.
You can add meta by adding an extra colon and appending the meta or a ? for any meta.
Example: 'dye:4', 'minecraft:dye:?', 'modname:item:meta'
These lists also support the OreDictionary using similar tags following the basic pattern oredict:orename. The oredict tag also supports meta just like the previous by appending a number or '?'
Example: 'oredict:dye:2' (So all dyes will meta 2) or 'oredict:dye:?' (So all dyes with any meta)
